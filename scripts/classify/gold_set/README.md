# Hand-labeling the gold set

`gold_set.yaml` is a stratified sample of papers used to validate
the LLM classifiers before bulk runs. Generated by:

```sh
./scripts/classify/cluster-papers
./scripts/classify/sample-gold        # writes gold_set.yaml here
```

## What you do

1. Open `gold_set.yaml`.
2. For each paper, read the `title` and `abstract` fields.
3. Replace every `?` under `labels:` with the correct value from
   the axis vocabulary.
4. Use `n-a` if the axis genuinely does not apply.
5. Use `unknown` only if **you** can't tell from the abstract — it
   sets the bar the LLM has to clear.

Open `src/precis/data/axes/<axis>.yaml` if you forget a vocabulary.

## Time budget

~30 papers × 7 axes × 30 sec = ~1 hour. Don't spend more than 60s
per paper; if it's that ambiguous, `unknown` is the right answer.

## After labeling

Run the eval harness (built next):

```sh
./scripts/classify/eval-classifier --axis scale
./scripts/classify/eval-classifier            # all axes
```

The harness calls the LLM on every paper in the gold set and
prints accuracy + confusion matrix per axis. Anything below 85%
needs prompt tuning before bulk run.

## Re-generating the sample

If you change the cluster count, axis set, or `n`, regenerate:

```sh
./scripts/classify/sample-gold --n 50 --seed 42
```

This **overwrites** any existing labels. To keep your work, copy
`gold_set.yaml` aside first.
