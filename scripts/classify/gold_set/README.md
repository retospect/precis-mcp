# Labeling the gold set

Two gold sets validate the classifiers before bulk runs:

- `gold_set.yaml` — one label per **paper** (ref axes). Generated by:
  ```sh
  ./scripts/classify/cluster-papers
  ./scripts/classify/sample-gold        # writes gold_set.yaml here
  ```
- `gold_set_chunks.yaml` — one label per **chunk** (`role`,
  `open-question`; ADR 0047). Generated by:
  ```sh
  ./scripts/classify/sample-gold-chunks # writes gold_set_chunks.yaml here
  ```

## Labeling: by hand or LLM-pre-label + adjudication

You may fill every `?` by hand (below), OR pre-label with an LLM and
then only adjudicate the contested rows — cheaper for the 200-chunk
set. Both gold sets were pre-labeled this way: each row carries
`*_conf` / `note` beside its value; the low-confidence rows were then
worked through and resolved (see `RESOLVED.md`). Pre-labels are NOT
truth until adjudicated.

## Accepted alternatives (`accept` / `role_accept`)

Some items are genuine coin-tosses (result vs interpretation; synthesis
vs experimental). Rather than force one side, the gold row carries an
**accepted-alternative** and the eval harness grades a prediction
correct if it equals the primary OR is in the accept list:

- chunks: `role_accept: [interpretation]` (and rarely `oq_accept: [..]`)
- papers: `accept: {studytype: [mixed], property: [electrical]}`

The **production** classifier still emits exactly one value per axis;
`accept` only affects scoring, so a defensible second choice isn't
penalised. The primary value is still the single best label.

## What you do

1. Open `gold_set.yaml`.
2. For each paper, read the `title` and `abstract` fields.
3. Replace every `?` under `labels:` with the correct value from
   the axis vocabulary.
4. Use `n-a` if the axis genuinely does not apply.
5. Use `unknown` only if **you** can't tell from the abstract — it
   sets the bar the LLM has to clear.

Open `src/precis/data/axes/<axis>.yaml` if you forget a vocabulary.

## Time budget

~30 papers × 7 axes × 30 sec = ~1 hour. Don't spend more than 60s
per paper; if it's that ambiguous, `unknown` is the right answer.

## After labeling

Run the eval harness (built next):

```sh
./scripts/classify/eval-classifier --axis scale
./scripts/classify/eval-classifier            # all axes
```

The harness calls the LLM on every paper in the gold set and
prints accuracy + confusion matrix per axis. Anything below 85%
needs prompt tuning before bulk run.

## Re-generating the sample

If you change the cluster count, axis set, or `n`, regenerate:

```sh
./scripts/classify/sample-gold --n 50 --seed 42
```

This **overwrites** any existing labels. To keep your work, copy
`gold_set.yaml` aside first.
